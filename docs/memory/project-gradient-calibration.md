The NowTrendIn gradient/detection score calibration (engine at `C:\Users\acinv\OneDrive\Desktop\CODING PROJECTS\NowTrendin`, deployed to Heroku `nowtrendin`). Built 2026-06-15. Determines "what is mainstream right now" via deviation from each topic's own baseline. Four layers:

1. **Dual-pathway detection** (`dual_pathway.py`): expert-origin topics use the existing gradient (niche/expert concentration = lead-time/runway — the moat, UNCHANGED); mainstream-origin topics use attention MAGNITUDE (real views/traffic). Blended by mainstream weight `w`. Wired in `score_all_topics` AFTER `apply_calibration` (calibration recomputes detection and would otherwise overwrite the blend). Gated by `DUAL_PATHWAY` env (default on).

2. **Baseline-relative (fame vs. diffusion)**: mainstreaming = deviation from the topic's OWN baseline breadth+magnitude, NOT absolute — so a household name's ambient fame (SpaceX always broad) reads neutral, and only expansion above its norm counts. Cold start (< `MIN_BASELINE_CYCLES`=3 history) falls back to absolute magnitude, flagged "calibrating". Baselines = median of last 12 cycles of stored `attention_magnitude` + `n_mainstream_platforms` columns (live-migrated into `velocity_scores`).

3. **Community-level tiers** (`community_tiers.py`): tier is a property of the COMMUNITY not the platform — SpaceX on r/SpaceX = expert, on r/all = mainstream. Applied at collection (Lemmy live; reddit/discourse ready). General social (bluesky/lemmy/mastodon) is mainstream unless a specialist community.

4. **Tier-migration velocity**: breadth counts distinct mainstream COMMUNITIES (source_name); a topic in expert communities now expanding into mainstream = crossing tiers = highest-confidence mainstreaming, amplifies `w`.

5. **News-outlet corroboration** (added 2026-06-15 sess.2, `dual_pathway.py`): news `engagement_raw≈log1p(120)=4.79` is BELOW the magnitude floor (7.0), so a pure-news topic registered magnitude≈0 and cold-start hard-zeroed breadth → multi-outlet news was invisible. Fix: count distinct reputable NEWS OUTLETS (`news_outlets`); `NEWS_MAINSTREAM_MIN=3`⇒`mainstream_confirmed` (absolute "arrived", distinct from baseline-relative `w`="moving"); cold-start uses ABSOLUTE breadth; `mainstream_detection` gains a cross-outlet term (×28). Persisted+surfaced columns: detection_pathway, mainstream_ratio, mainstream_breadth, news_outlets, mainstream_confirmed, tier_migration. Backtest now 13 cases.

**Data-quality layer (2026-06-15 sess.2):** (a) **common-word filter** — `common_words.txt` (~10.3k frequent English words via wordfreq, proper nouns subtracted; regen with `_gen_common_words.py`); `_is_quality_topic` rejects any single common word regardless of volume (suffering/saying/exclusive out; japan/fifa/apple kept); applies at extraction AND serve-time. (b) **consolidation** — `_canonicalize_topic` folds variants (demonym→country japanese→japan, possessive, alias gpt→chatgpt) in `_topic_key`; `_consolidate_window_keys` self-heals existing pool each score cycle + deletes orphaned non-canonical velocity_scores rows. (c) **source-aware explainer** — `_topic_source_context()` feeds real headlines+platforms to `ai_grade.explain_topic(topic, context=)`. (d) **AI cost cap** — unified `ai_costs` ledger + hard `AI_MONTHLY_BUDGET_USD` (default $20) gating explainer/backfill/grade; `GET /ai/costs`. Remaining follow-up: deeper entity resolution (trump vs Donald Trump).
